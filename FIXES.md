## 13. เปิดไฟล์แล้วเงียบสนิททั้งไฟล์ (playhead ไม่ขยับด้วย)

- **อาการ:** บางเซสชันกดเล่นแล้วไม่มีเสียงเลยกับทุกท่า (เจอกับ story.wav)
- **สาเหตุ:** WebKit ระงับ (suspend) AudioContext ที่ถูกสร้างก่อนผู้ใช้คลิกครั้งแรก
  และ WebAudioPlayer ของ wavesurfer **ไม่เคยเรียก resume() เลย** → เสียงเงียบ
  และนาฬิกา context หยุดเดิน (เลยดูเหมือนแอปค้าง) — อาการเกิดๆ หายๆ ตามจังหวะ
  ว่า context ถูกสร้างก่อนหรือหลัง gesture แรก
- **วิธีแก้:** `ensureAudioRunning()` ใน `src/audio/player.ts` — เช็ค
  `audioContext.state === "suspended"` แล้ว `resume()` ในทุกทางเข้าของการเล่น
  (playPause / playFrom / playRange ล้วนถูกเรียกจาก user gesture อยู่แล้ว)
- **กันเกิดซ้ำ:** เพิ่มทางเข้าการเล่นใหม่เมื่อไหร่ ต้องเรียก ensureAudioRunning ก่อน
- **⚠ รอบ 2 (เสียงหายซ้ำแม้มี resume):** WebAudio ของ WebKit ดับเองได้ทั้งที่สถานะ
  "running" (เช่น หลังสลับอุปกรณ์เสียง) → **เลิกใช้ WebAudio เล่นเสียงถาวร**:
  ถอดรหัสครั้งเดียว → encode เป็น WAV (src/audio/wav.ts) → เล่นผ่าน media element
  ธรรมดาซึ่งเสถียรใน WebKit; ภาพ/เสียง sync กันโดยโครงสร้างเพราะมาจาก buffer
  ถอดรหัสชุดเดียวกัน (คงหลักการข้อ #7) — **กติกาใหม่: playback ต้องกินข้อมูล
  PCM ชุดเดียวกับที่ใช้วาดภาพเสมอ ไม่ว่าจะเล่นผ่านกลไกไหน**

## 20. หลังใช้ค้นหา/Tab แล้ว "คลิกคำไหนก็เงียบ" ไม่ไฮไลต์เสียงอีกเลย (edit-box latch)

- **อาการ:** ใช้ค้นหาแล้วกดคำที่เจอ → ไม่เด้งไปที่คำ ไม่ไฮไลต์เสียง และหลัง
  จากนั้นคลิกคำไหนก็เงียบทั้งไฟล์ จนต้องเปิดไฟล์/แอปใหม่
- **สาเหตุ:** กล่องแก้คำ (input) อาศัยอยู่ "ใน" span ของคำ และปิดตัวด้วย
  blur/Enter เท่านั้น — แต่ `refresh(i)` เขียน `span.textContent = ...` ซึ่ง
  **ทำลาย input โดย blur ไม่ยิง** (ลบ element ที่โฟกัสอยู่ browser ไม่ส่ง
  blur) → ธง `editing` ค้าง true ตลอดกาล → ทุกคลิก/ดับเบิลคลิกโดนประตู
  `if (editing) return` กลืนหมด; มีหลายทางที่ refresh โดน span ที่กล่องเปิด:
  ไฮไลต์คำตามเสียง (Tab เปิดกล่อง+เล่นเสียง → เสียงพ้นคำ → refresh),
  ผลค้นหา (setSearchMatches), ซ่อน filler (refreshAll)
- **วิธีแก้ (3 ชั้น ใน transcript.ts):** (1) `refresh()` ห้ามทับ span ที่มีกล่อง
  เปิด (2) ประตู editing ทุกจุดใช้ `editingActive()` ที่เช็คว่า editor element
  ยังอยู่ใน DOM จริง — หายไป = ปลดธงเอง (self-healing) (3) `render()/clear()`
  รีเซ็ตธงเสมอ
- **กันเกิดซ้ำ:** UI ที่ "ฝังใน DOM ที่คนอื่น repaint ได้" ห้ามถือ state ด้วย
  ธงลอยๆ — ต้องผูกกับ element จริง (isConnected) และ repaint ต้องเว้นจุดที่
  กำลังแก้; ทดสอบ: Tab เปิดกล่อง → ปล่อยเสียงเล่นจนพ้นคำ → กล่องต้องยังอยู่
  และพิมพ์ต่อได้
